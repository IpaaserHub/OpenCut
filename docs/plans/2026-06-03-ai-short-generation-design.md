# AIショート動画 自動生成 設計書

作成日: 2026-06-03
対象リポジトリ: `IpaaserHub/OpenCut`（SNSDir共通動画エディタ / OpenCut-app/OpenCut の fork）
ステータス: 設計合意済み（MVP着手前）

---

## 1. 目的

長尺動画（例: 1時間）を入れると、AIが内容を理解し、ユーザーが指定した尺（15秒/30秒/1分）と構成テンプレートに沿って、**自動でショート動画を切り貼り＋テロップ付与して1本生成する**機能を作る。

YTDir / TKDir など各 SaaS が iframe で読み込む共通エディタ上で動作する。

## 2. スコープ

### MVP（今回）

- 入力材料は **文字起こしテキストのみ**（映像解析は後フェーズ）
- 構成は **ユーザーがプリセットを選ぶ → AIが当てはめる**
- 範囲は **自動カット + テロップ自動配置** まで（BGM/SEは後フェーズ）
- 並び順の **入れ替えOK**（結論先出し等を可能にする）
- 尺は **プリセット + 微調整**（±数秒）
- 1回の実行で **1本** 出力

### MVPに含めないもの（後フェーズ）

- 映像/音量解析、BGM/SE自動付与、複数バージョン同時出力、ユーザー自作テンプレ、テナント別ノウハウ反映

---

## 3. 全体パイプライン

```
[長尺動画を読込]
      ↓
[文字起こし] ← 既存 services/transcription をそのまま利用
      ↓ segments: { text, start, end }[]
[AI呼び出し] ← サーバー側（Claude API 想定。APIキーをブラウザに晒さない）
   入力: テンプレ種類 / 目標尺 / セグメント一覧
      ↓
[構成プラン] = 選択セグメント + 並び順 + 各テロップ文 + 冒頭フック文 + CTA文
      ↓
[エディタが自動組み立て] ← 選択区間をタイムラインにクリップ配置 + テロップ
                            （テロップは既存「テキストテンプレート」機能を流用）
      ↓
[ユーザーが微修正 → 書き出し]
```

### 既存資産との接続

| 工程 | 利用する既存資産 |
|---|---|
| 文字起こし | `apps/web/src/services/transcription/`（Whisper系。`{ text, start, end }` 出力） |
| テロップ | 本ブランチで実装中の「テキストテンプレート」機能 |
| タイムライン配置 | 既存のクリップ/トラック配置ロジック |

### 新規に必要なもの

1. **サーバー側のAI呼び出しエンドポイント**（文字起こしと違い、LLM呼び出しは鍵を要するためサーバー経由）
2. **構成プラン → タイムライン組み立て**のロジック
3. **プリセット定義（指示書）** のデータ

## 4. 設計の核: テンプレートを3部品に分解

短尺動画ノウハウ記事7本（note/goodcomms, boosttech, jarea, prx-studio-q, excrie, nikkei xtrend, tenani, buzz-brain, dsp）を突き合わせた結果、どの記事も同じ3層構造を指していた。フック・本編・CTAは**独立して差し替え可能な部品**。

```
① フック(冒頭0〜3秒) … 視聴維持率を最も左右（全記事が最重要と明言）
② 本編フレーム       … 1動画1メッセージ / 2〜3秒ごとに変化
③ CTA(ラスト3秒)     … フォロー・保存・誘導・次回予告
```

**テンプレート = ① × ② × ③ の組み合わせプリセット。**
ユーザーには完成形プリセットを名前で見せ、裏は部品の組み合わせ。新テンプレ追加が安く、Phase 4-5（ユーザーカスタマイズ / テナント別）の土台になる。

### ① フック部品（全て文字起こしで生成可）

| フック | 例 |
|---|---|
| 衝撃の事実型 | 「○○知らないのは損」「○○の90%は間違い」 |
| 疑問提起型 | 「なぜ○○は△△なのか?」「あなたの○○大丈夫?」 |
| 数字訴求型 | 「売上2倍になった方法を30秒で」 |
| ベネフィット提示型 | 「これで○○が解決します」 |
| 意外性・逆張り型 | 常識を覆す / 予想外 |
| 共感(あるある)型 | 「○○あるある」 |
| ビフォーアフター見せ型 | 完成・結果を先に出す（一部映像要） |

### ② 本編フレーム部品

| フレーム | 流れ | 文字起こしのみ |
|---|---|---|
| 解説型 | 要点を分かりやすく | ✅ |
| 課題解決型(PASONA) | 課題→解決→効果 | ✅ |
| 結論先出し(PREP) | 結論→理由→例→結論 | ✅ |
| リスト/○○3選型 | 列挙・カウントダウン | ✅ |
| 比較型(○○の違い) | 対立構造 | ✅ |
| クイズ・問いかけ型 | 問い→答え | ✅ |
| スゴ技/裏ワザ型 | 知って得する技 | ✅ |
| ストーリーテリング型 | 導入→展開→オチ | ✅ |
| 要点まとめ型 | 長尺を圧縮 | ✅ |
| 検証してみた型 | 企画→実施→反応 | 🎬 |
| ビフォーアフター型 | 現状→過程→結果 | 🎬 |
| リズム/音ネタ型 | 替え歌・ダンス | ✘（流用不可）|

### ③ CTA部品（全て文字起こしで生成可）

フォロー促進 / 保存促進（「後で見返す人は保存」）/ プロフィール・LINE誘導（最重要）/ 次回予告

---

## 5. MVPプリセット（5つ）

各プリセットは ①フック × ②本編 × ③CTA の組み合わせとして定義。全て文字起こしのみで生成可・テロップ主体・映像解析不要。

| プリセット | ①フック | ②本編 | ③CTA |
|---|---|---|---|
| 結論先出し型 | 衝撃の事実 | PREP(結論→理由) | 保存促進 |
| 問題提起→解決型 | 疑問提起 | 課題解決(PASONA) | プロフィール誘導 |
| ○○3選/リスト型 | 数字訴求 | リスト列挙 | フォロー促進 |
| How-to手順型 | ベネフィット提示 | 手順解説 | 保存促進 |
| 要点まとめ型 | 疑問提起 | 要点圧縮 | 次回予告 |

---

## 6. 全プリセット共通の制約（AIに必ず守らせる）

1. **冒頭0〜3秒が命** — フックを外すと離脱。最優先。
2. **1動画1メッセージ** — テーマを1つに絞る。
3. **2〜3秒ごとに視覚変化** / 1カット2〜3秒目安。
4. **尺の目安**: TikTok 15〜30秒 / YTショート 30〜45秒。
5. **ラスト3秒にCTA**。

---

## 7. AIコントラクト（入出力・案A=セグメント選択方式）

採用方式は **案A: セグメント選択方式**。AIにセグメント一覧を渡し、選んだ番号・並び順・テロップを返させる。元動画の時刻に確実に対応が取れ、切り出し位置がズレない。検証も速い。
（不採用の案B=台本先行方式は、台本と文字起こしの照合事故が起きやすくMVPに不向き。）

### 入力（サーバーへ送る）

```jsonc
{
  "presetId": "conclusion-first",     // 選択プリセット
  "targetSeconds": 30,                 // プリセット + 微調整後の目標尺
  "segments": [                        // 文字起こし結果（番号付与）
    { "index": 0, "start": 0.0, "end": 3.2, "text": "..." },
    { "index": 1, "start": 3.2, "end": 7.8, "text": "..." }
    // ...
  ]
}
```

### 出力（AIが返す構成プラン）

```jsonc
{
  "hookText": "知らないと損する○○",       // 冒頭テロップ（元動画に無い文言可）
  "clips": [
    { "segmentIndex": 12, "order": 0, "caption": "結論はこれ" },
    { "segmentIndex": 3,  "order": 1, "caption": "理由その1" }
    // 並び順入れ替え可（order で表現）
  ],
  "ctaText": "後で見返す人は保存",          // ラストテロップ
  "estimatedSeconds": 29                    // 合計尺の見積もり
}
```

- 出力は構造化スキーマで強制（バリデーションで不正出力をリトライ）。
- `segmentIndex` は入力の `index` に対応 → そのまま `start`/`end` を引いてタイムライン配置。

---

## 8. エラーハンドリング / エッジケース

| ケース | 対応 |
|---|---|
| AIが存在しない segmentIndex を返す | バリデーションで弾いてリトライ／該当クリップを除外 |
| 合計尺が目標を大幅超過/不足 | 超過分は末尾から削る／不足は隣接セグメントを足す。許容±数秒。 |
| 文字起こしが空 or 極端に短い | 「素材が短すぎます」とユーザーに通知し生成中止 |
| 並び替えで話が繋がらない | MVPは許容（ユーザーが微修正）。Phase 3で接続スコアリング検討 |
| AI API 障害 / タイムアウト | エラー表示し、再試行ボタン。文字起こし・手動編集は独立して動作 |
| 長尺すぎてトークン超過 | セグメントを間引き or 分割して要約してから渡す（要設計） |

---

## 9. テスト方針

- **ユニット**: 構成プラン → タイムライン組み立て（segmentIndex → クリップ変換、order での並び替え、尺超過時のトリム）。AI出力はモックで固定。
- **スキーマ検証**: 不正な AI 出力（範囲外index、空clips）をリトライ/除外できるか。
- **既存の transcription テスト**（`transcription/__tests__/caption.test.ts`）を壊さないこと。
- AIの「内容理解の質」自体は自動テスト困難 → 実動画での目視QA（プリセット別に数本）。

## 11. プリセット実装バックログ（候補の棚）

### Phase 2候補：文字起こしのみで作れる ✅

| プリセット候補 | ①フック | ②本編 | ③CTA |

| 比較型(A vs B) | 逆張り | 対比 | フォロー |
| クイズ・問いかけ型 | 疑問提起 | 問い→答え | コメント誘導 |
| スゴ技・裏ワザ型 | 衝撃の事実 | 技の手順 | 保存 |
| あるある共感型 | 共感 | あるある列挙 | フォロー |
| 誤解を正す型(実は〜) | 意外性 | 通説否定→正解 | 保存 |
| 体験談・ストーリー型 | 共感 | 起承転結 | プロフィール誘導 |
| 名言・パンチライン型 | 衝撃 | キメ台詞連結 | フォロー |
| ティーザー(焦らし)型 | おいしい所チラ見せ | 本編 | 次回予告 |
| TOP5カウントダウン型 | 数字訴求 | 逆順列挙 | 最後まで誘導 |

### Phase 3候補：映像/音量解析が要る 🎬

| 候補 | 必要な解析 |
|---|---|
| 名場面ハイライト型（盛り上がり） | 音量・歓声・笑いのピーク検出 |
| リアクション・表情型 | 顔アップ・驚き顔検出 |

| テンポカット最適化 | 無音・間を波形で自動カット |
